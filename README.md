## How It Works

### 1. Task Intake
Tasks are ingested, validated, and assigned a unique ID for traceability.

### 2. Governance Rules Application
Deterministic rules classify the task and route it to appropriate agent pool.

### 3. Agent Selection & Assignment
Agents are selected based on capability, load, and governance rules.

### 4. Execution & Monitoring
Agent executes task while system monitors for anomalies and rule violations.

### 5. Outcome Audit & Capsule Creation
Outcome is recorded, and successful patterns are stored in SHA-384 memory capsules.

### 6. Compliance Recording
Complete audit trail recorded for regulatory review.
